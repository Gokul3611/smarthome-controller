# Release APK Directory

This directory contains release builds of the Smart Home Controller Android app.

## Naming Convention

Release APKs should follow this naming pattern:
```
smart-home-controller-v{major}.{minor}.{patch}-release.apk
```

Examples:
- `smart-home-controller-v1.0.0-release.apk`
- `smart-home-controller-v1.0.1-release.apk`
- `smart-home-controller-v2.0.0-release.apk`

## Building Release APK

From the Flutter project directory:
```bash
cd "frontend/flutter project/demo"
flutter build apk --release
cp build/app/outputs/flutter-apk/app-release.apk ../../../app/release/smart-home-controller-v1.0.0-release.apk
```

## APK Characteristics

Release APKs are:
- ✅ Optimized for production use
- ✅ Smaller file size
- ✅ Code obfuscation enabled
- ✅ Should be signed with release keystore
- ❌ No debug symbols included
- ❌ Cannot be debugged

## Distribution

Release APKs can be distributed via:
- GitHub Releases
- Direct download links
- App stores (after proper signing)
- Internal distribution platforms

---

**Note:** APK files are not committed to the repository. This directory exists to provide a consistent location for placing built APKs.
